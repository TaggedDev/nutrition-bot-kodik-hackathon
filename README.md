# Nutrition Bot

**AI-инструмент для быстрого и удобного логирования питания**

Проект создан в рамках хакатона Kodik IDE как персональный AI-ассистент, который помогает пользователю фиксировать приёмы пищи быстрее, чем в классических calorie-tracking приложениях. Вместо ручного поиска продуктов и отдельных полей ввода пользователь может описать еду в свободной форме, получить структурированный результат с КБЖУ и сохранить запись в дневник.

---

## 1. Что это за проект

Nutrition Bot решает простую, но болезненную задачу: сделать учёт питания достаточно быстрым, чтобы человек не бросал его через несколько дней. В обычных приложениях пользователь тратит время на поиск нужного продукта, выбор карточки, корректировку порции и ручную проверку макронутриентов. Здесь эта цепочка сокращается до короткого запроса в интерфейсе, близком к чату.

Продукт ориентирован на сценарий, где человеку важно быстро понять, что он съел, сколько это дало калорий, белков, жиров и углеводов, и как это вписывается в дневной рацион. Основная идея MVP: собрать пайплайн, где пользователь общается с агентом и получает структурированную формализированную запись о приеме пищи.

---

## 2. Бизнес-ценность

С точки зрения пользователя продукт снижает главный барьер food tracking-сервисов: высокую стоимость ручного ввода. Чем меньше действий нужно для фиксации еды, тем выше вероятность регулярного использования и удержания.

С точки зрения рынка Nutrition Bot может быть полезен в нескольких сегментах:

- людям, которые следят за весом, калориями и макронутриентами;
- пользователям fitness и wellness-сервисов;
- digital-health продуктам, которым нужен удобный nutrition onboarding;
- B2C-сценариям подписочного сервиса с AI-ассистентом по питанию;
- B2B2C-интеграциям в фитнес-клубы, health-коучинг и программы сопровождения.

Бизнес-гипотеза проекта в том, что conversational UX и AI-assisted parsing заметно повышают конверсию в регулярное ведение дневника питания по сравнению с классическим интерфейсом таблиц и каталогов продуктов.

---

## 3. Что уже реализовано

В текущем MVP уже собран рабочий контур продукта:

- веб-приложение с chat-like интерфейсом для взаимодействия с nutrition-ассистентом;
- регистрация, вход, выход и получение данных текущего пользователя;
- сохранение записей по дню и типу приёма пищи;
- поиск nutrition-данных по текстовому запросу;
- поиск продуктов и nutrition-данных по штрихкоду;
- интеграция с Open Food Facts;
- confidence score для оценки качества найденного результата;
- rate limiting и кеширование для внешнего API;
- профиль пользователя и дневные данные по питанию;
- голосовой UI-элемент и подготовка интерфейса к мультимодальному сценарию;
- unit, integration и e2e-покрытие для ключевых пользовательских потоков.

Это уже позволяет показать реальный пользовательский сценарий: пользователь авторизуется, вводит описание еды, получает список найденных продуктов с КБЖУ и сохраняет запись в свой рацион.

## 5. Что будет дальше

Следующий этап развития проекта сфокусирован не на расширении количества экранов, а на усилении самого AI-сценария и продуктовой полезности:

- production-хранилище для meal-данных без mock-слоя;
- подтверждение и редактирование meal entry после AI-разбора;
- дневная сводка питания и прогресс по цели;
- рекомендации следующего приёма пищи на основе текущего дневного баланса;
- полноценный мультимодальный pipeline: voice -> STT, photo -> OCR, text -> structured nutrition JSON;
- более точный LLM-парсинг составных блюд и неоднозначных запросов;
- расширение источников данных помимо Open Food Facts;
- подготовка к production deployment и CI/CD.

Итоговая продуктовая цель: превратить Nutrition Bot из удобного логгера в полноценного AI-ассистента по питанию, который не просто считает КБЖУ, а помогает человеку держать курс на свою цель.

---

## 6. Технологический стек

- Backend: C#, ASP.NET Core 8 Web API
- Архитектура: Clean Architecture (Domain / Application / Infrastructure / Web / Shared)
- База данных: PostgreSQL
- ORM и миграции: Entity Framework Core
- Аутентификация: ASP.NET Core Identity, Cookie-based Auth
- Frontend: React 18, TypeScript, Vite
- Внешние источники данных: Open Food Facts
- AI-компоненты и agentic pipeline: Codex / Kodik IDE, DeepSeek, Tavily
- Контейнеризация: Docker, Docker Compose
- Тестирование: xUnit, интеграционные тесты, Playwright E2E

---

Проект разрабатывался с активным использованием Kodik IDE и AI-ассистента Codex: для проектирования архитектуры, генерации кода, ускорения интеграций, отладки инфраструктуры и развития MVP в короткий срок хакатона.
